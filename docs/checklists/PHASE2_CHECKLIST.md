# ✅ Phase 2 Checklist - 핵심 기능 개발

## 🏗️ 아키텍처 규칙 준수

- [ ] **Domain 레이어**

- [ ] **Application 레이어**
  - [ ] Use Cases 구현
  - [ ] Domain 인터페이스만 사용
  - [ ] DTO 정의

- [ ] **Infrastructure 레이어**
  - [ ] Domain 인터페이스의 구현체

  - [ ] 캐싱 로직

- [ ] **Presentation 레이어**
  - [ ] API 엔드포인트 구현
  - [ ] 요청/응답 처리
  - [ ] 에러 응답 형식화
  - [ ] Application 사용

- [ ] **의존성 방향**
  - [ ] 항상 외부 → 내부 방향
  - [ ] 역방향 의존 없음
  - [ ] 순환 의존 없음

## 🎯 SOLID 원칙

- [ ] **S: 단일 책임**
  - [ ] 각 클래스는 하나의 책임만
  - [ ] 변경 이유가 하나 이하

- [ ] **O: 개방-폐쇄**
  - [ ] 새 기능 추가 시 기존 코드 수정 없음
  - [ ] 인터페이스로 확장

- [ ] **L: 리스코프 치환**
  - [ ] 자식 클래스는 부모로 대체 가능
  - [ ] 계약 위반 없음

- [ ] **I: 인터페이스 분리**
  - [ ] 큰 인터페이스 분리
  - [ ] 필요한 메서드만 정의

- [ ] **D: 의존성 역전**
  - [ ] 구체 클래스가 아닌 인터페이스에 의존
  - [ ] 의존성 주입(DI) 사용
