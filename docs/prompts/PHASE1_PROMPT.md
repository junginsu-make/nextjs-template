# 역할
당신은 시니어 프로덕트 매니저이자 시스템 아키텍트입니다.

# 작업
다음 프로젝트의 **간소화된 초기 문서 3종**을 생성하세요:

## 프로젝트 정보
- 프로젝트명: [프로젝트명]
- 목적: [목적]
- 주요 기능: [기능 1, 2, 3...]

## 생성할 문서

### 1. PRD.md (2-3페이지 분량)
다음 섹션만 포함:
- **프로젝트 개요** (3-4문장)
- **비즈니스 목표** (3-5개 bullet points)
- **핵심 기능** (기능명 + 1문장 설명)
- **성공 지표** (3-5개 KPI)

❌ 포함하지 마세요: 상세 요구사항, API 명세, 화면 설계

### 2. LLD.md (2-3페이지 분량)
다음 섹션만 포함:
- **시스템 아키텍처** (다이어그램 + 간단한 설명)
- **기술 스택** (표 형식)
- **주요 컴포넌트** (이름 + 역할 1문장)
- **데이터 모델** (주요 엔티티만 ERD)

❌ 포함하지 마세요: 상세 API 설계, 컴포넌트 내부 로직, 상세 스키마

### 3. PLAN.md (5-7페이지 분량) ⭐ 가장 중요!
다음 형식으로 작성:

#### Phase 1: [Phase 이름]
**목표**: [구체적 목표]
**기간**: [예상 기간]

**구현 작업**:
1. [작업 1]
   - 상세 내용
   - 예상 시간
2. [작업 2]
   ...

**산출물**:
- [ ] 코드: [파일명]
- [ ] 문서 업데이트: PRD.md 섹션 [X], LLD.md 섹션 [Y]

**검증 기준**:
- [ ] [테스트 항목 1]
- [ ] [테스트 항목 2]

#### Phase 2: ...
[같은 형식 반복]

```markdown

```
```markdown
// LLD.md 내용
```
