## 🎯 Phase 2 작업

**PLAN.md의 Week 2-3 작업을 순서대로 구현해주세요.**

각 Day 작업마다 다음 순서로 진행:

### **1단계: PLAN.md 확인**
- 오늘 할 Day {N} 작업 확인
- 예상 시간, 완료 기준 확인

### **2단계: 코드 구현**
- PLAN.md에 명시된 파일 생성
- 헥사고날 아키텍처 준수
- SOLID 원칙 준수

### **3단계: 문서 업데이트** (중요!)
PLAN.md에 명시된 대로 PRD/LLD 업데이트:

**PRD.md 업데이트**:
- 섹션: "{PLAN에 명시된 섹션}"
- 내용: "상세 기능 명세 추가"

**LLD.md 업데이트**:
- 섹션: "{PLAN에 명시된 섹션}"
- 내용: "DB 스키마/API 명세 추가"

### **4단계: 상세 설계 문서 생성** (선택)
- 파일: `docs/phases/phase-2-day-{N}.md`
- 내용: 해당 Day의 상세 설계 (API 명세, 컴포넌트 구조 등)

### **5단계: Git 커밋**
```bash
git add .
git commit -m "feat: Week 2 Day {N} - {작업 내용}

- {세부 사항 1}
- {세부 사항 2}
- PRD/LLD 업데이트"
git push
```

### **6단계: PLAN.md 체크박스 업데이트**
- [ ] → [x] 완료 표시
- 진행률 바 업데이트
